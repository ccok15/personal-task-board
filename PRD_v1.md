# 个人测试任务看板网站 PRD V1

## 1. 产品定位
- 面向芯片硬件测试工程师的轻量任务终端。
- 核心目标：把“当前在做什么”和“全部任务如何检索查看”统一到一个站点里。
- V1 只保留两条主线：`当前任务首页`、`任务管理页`，辅以 `管理员后台任务编辑`。

## 2. 目标用户
- **公开访客 / 合作方**：查看公开任务、在首页新建任务、把公开未完成任务改为搁置。
- **管理员**：查看公开 + 私人任务、维护全部字段、修改全部状态。

## 3. 数据模型
### Task
- 用于统一表达合作方提交任务、管理员手工录入任务、当前正在处理的工作内容。
- 字段：
  - `title`：标题，必填
  - `description`：描述，可选
  - `submitterName`：提交人，必填
  - `priority`：`URGENT | HIGH | MEDIUM | LOW`
  - `status`：`PENDING | IN_PROGRESS | BLOCKED | PAUSED | DONE`
  - `dueDate`：期望完成时间，可选
  - `completedAt`：完成时间，仅已完成任务使用
  - `adminNote`：管理员备注，仅后台可见
  - `isPublic`：是否公开

### User
- V1 仅预置一个管理员账号。
- 继续保留独立 `User` 表，为未来多管理员扩展做准备。

## 4. 前台行为
- 首页 `/`
  - 展示全部未完成任务：`PENDING / IN_PROGRESS / BLOCKED / PAUSED`
  - 顶部提供唯一的新建任务入口
  - 游客和管理员都可在首页创建任务
  - 游客只能把公开未完成任务改为 `PAUSED`
  - 管理员可在首页把任务直接改成任意状态
  - 管理员在首页能看到公开 / 私人标识，并可进入编辑页
- 任务管理页 `/tasks`
  - 仅游客使用
  - 展示全部公开任务
  - 只提供搜索查看，不提供编辑入口

## 5. 后台行为
- `/admin` 直接跳转到 `/admin/tasks`
- `/admin/tasks`
  - 查看全部任务
  - 搜索、筛选、区分公开 / 私人
  - 快速标记任务完成
  - 进入任务编辑页
- 管理员不使用 `/tasks` 已完成页，访问时直接跳转到 `/admin/tasks`
- `/admin/tasks/[id]`
  - 管理员可编辑标题、描述、提交人、优先级、状态、截止时间、公开性、管理员备注
- 新建任务不再通过后台单独页面完成，统一回到首页

## 6. 路由与权限
- `/submit` 重定向到 `/?create=1`
- `/plans` 与 `/admin/plans*` 重定向到 `/`
- 游客永远看不到私人任务
- 公开页面上不显示“公开任务”字样，统一只叫“任务”

## 7. UI 规范
- 风格：科幻控制台，不依赖贴图素材
- 表达方式：
  - 深色背景
  - 青蓝 / 绿色高亮
  - 状态徽章、优先级颜色、公开 / 私人标识
  - 轻动效，仅用于悬浮、切换、状态反馈

## 8. 非目标
- 不做计划模块
- 不做邮件通知
- 不做验证码 / 限流 / 反滥用
- 不做评论、文件上传、第三方登录
- 不做合作方自助编辑或撤回任务
- 不做多管理员界面

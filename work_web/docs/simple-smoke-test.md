# 极简冒烟测试清单

适用范围：每次修改**小功能**后执行一次。  
目标：只检查有没有明显的大问题，不做全量测试，不截图。

## 执行规则

- 每次小改动后跑 1 遍
- 只测主流程，控制在 2 到 3 分钟内
- 如果出现 `500`、白屏、拒绝访问、死循环跳转，就先修复再继续

## 测试步骤

### 游客

1. 打开首页 `/`
   - 页面能打开
   - 任务列表能显示
2. 打开任务管理 `/tasks`
   - 页面能打开
   - 搜索框和列表能显示

### 管理员

1. 打开登录页 `/login`
   - 页面能打开
2. 使用管理员账号登录
   - 能进入 `/admin/tasks`
3. 打开首页 `/`
   - 页面能打开
   - 首页任务列表能显示
4. 退出登录
   - 能回到前台页面

## 通过标准

- 游客和管理员都能正常进入各自页面
- 没有明显报错、白屏、拒绝访问
- 主要按钮和列表能正常显示

## 记录格式

- 日期
- 修改内容
- 结果：通过 / 失败
- 备注：只写一句问题描述或“无”
